# 小房东/合租房一页式租金、维修、票据管家开发手册

## 1. 产品定位

### 1.1 一句话说明

面向只有 1-5 套房或几个房间出租的小房东，提供一个按房源、房间、租客和月份自动归档租金、押金、维修、票据、合同，并在年底导出税务包的轻量工具。

### 1.2 第一版聚焦

第一版只做法国市场，优先服务法国 LMNP 小房东。

第一版语言只完整提供法语 `fr`，但代码、路由、数据库和文案结构必须从一开始支持多语言和多国家扩展。

### 1.3 核心用户

- 拥有 1-5 套出租房的小房东。
- 合租房房东，按房间出租。
- LMNP 房东。
- 同时存在长期租赁、短租、合租的个人业主。
- 当前用 Excel、Google Drive、邮箱、WhatsApp、照片相册保存票据的人。

### 1.4 核心痛点

- 房租、押金、维修、票据、合同散落在不同地方。
- Excel 公式容易坏，年底汇总麻烦。
- 收据和发票没有按房源、月份、税务类别归档。
- 报税时不知道哪些费用能扣、票据是否齐。
- 传统物业软件太重，面向企业或多房源管理，不适合小房东。

### 1.5 MVP 成功标准

用户能在 30 分钟内完成：

1. 创建一个房源。
2. 创建房间或整租单元。
3. 添加租客和租期。
4. 录入 3-12 个月租金。
5. 上传维修票据或费用发票。
6. 查看年度概览。
7. 导出一个税务准备包 ZIP 或 PDF。

## 2. 产品结构

### 2.1 推荐第一版架构

第一版使用一个 Next.js 项目，路由如下：

```text
/                 落地页
/login            登录/注册
/dashboard        产品后台首页
/properties       房源列表
/properties/[id]  房源详情
/tenants          租客列表
/documents        票据与合同中心
/tax              年度税务包
/settings         设置/账单/国家与语言
```

后续品牌变强后再拆分：

```text
yourdomain.com       落地页
app.yourdomain.com   产品后台
```

第一版先不拆子域名，降低部署和认证复杂度。

### 2.2 首次打开与国家选择

建议注册时绑定国家，而不是第一次打开网站就强制选择。

原因：

- 落地页可以根据浏览器语言展示默认文案，不需要马上知道国家。
- 税务分类、币种、日期格式、导出模板都和账户强相关，应该绑定在用户 workspace/profile 上。
- 后续用户可能拥有跨国房源，但第一版不支持，避免过早复杂化。

第一版规则：

- 注册流程中必须选择 `country`。
- 默认国家为 `FR`。
- 第一版 UI 可显示国家选择，但只启用 France。
- 数据库字段从第一天保留 `country_code`。
- 税务模板根据 `country_code + tax_regime` 选择。

后续扩展：

- `user_profile.country_code` 表示账户默认国家。
- `properties.country_code` 表示单个房源所在国家。
- 第一版可保持二者一致。

### 2.3 多语言策略

使用 `next-intl`。

目录结构：

```text
messages/
  fr.json
  en.json
  zh.json
```

第一版只完整维护：

```text
messages/fr.json
```

`en.json` 和 `zh.json` 可以只保留最小占位，避免构建失败。

建议 locale 规则：

- 默认 locale: `fr`
- 支持 locale: `fr`, `en`, `zh`
- 第一版公开产品以法语为主。
- 用户设置里保留语言字段 `locale`。

推荐文案命名：

```json
{
  "common": {
    "save": "Enregistrer",
    "cancel": "Annuler"
  },
  "landing": {
    "heroTitle": "Vos loyers, reçus et travaux prêts pour la déclaration"
  },
  "dashboard": {
    "title": "Tableau de bord"
  }
}
```

## 3. MVP 功能范围

### 3.1 必做功能

#### 账户与设置

- 邮箱注册/登录。
- 忘记密码。
- 用户 profile。
- 国家选择：第一版只允许 France。
- 语言选择：第一版主要 French。
- 账单入口：Stripe Customer Portal。

#### 房源管理

- 创建房源。
- 房源字段：
  - 名称。
  - 地址。
  - 国家。
  - 城市。
  - 出租类型：整租、合租、短租混合。
  - 税务制度：第一版 `LMNP`。
- 房间/单元管理：
  - 整套出租时可自动创建一个默认单元。
  - 合租时可创建多个房间。

#### 租客管理

- 创建租客。
- 关联到房源/房间。
- 租期开始和结束日期。
- 月租金额。
- 押金金额。
- 租客状态：当前、已退租。

#### 租金记录

- 按月份查看应收和实收。
- 手动标记已收。
- 支持部分付款。
- 支持备注。
- 支持上传付款凭证。

第一版不接入银行，不做自动收款。

#### 费用与维修

- 添加费用记录。
- 上传发票/收据图片或 PDF。
- 费用关联到房源、房间或全局账户。
- 字段：
  - 日期。
  - 金额。
  - 商家/供应商。
  - 税务分类。
  - 是否有票据。
  - 文件附件。
  - 备注。

#### 文件与票据中心

- 上传文件。
- 文件类型：
  - 租赁合同。
  - 租金收据。
  - 维修发票。
  - 保险。
  - 贷款/利息文件。
  - 税务文件。
  - 其他。
- 文件必须可关联：
  - 房源。
  - 房间。
  - 租客。
  - 月份。
  - 费用记录。

#### 年度税务包

- 按年份选择。
- 汇总：
  - 年租金收入。
  - 押金记录。
  - 可扣费用总额。
  - 按类别的费用明细。
  - 缺失票据提醒。
- 导出：
  - CSV 明细。
  - PDF 汇总。
  - ZIP 文件包，包含 PDF、CSV、原始票据文件。

#### 订阅/付费

第一版推荐：

- 免费试用或免费导出水印版。
- 一次性买断：`29-59 EUR`。
- 或低价订阅：`5 EUR/month`。

MVP 技术上先用 Stripe Checkout + Customer Portal。

### 3.2 暂不做功能

- 银行自动同步。
- 租金在线支付。
- 自动生成正式报税申报表。
- 多成员团队权限。
- 移动 App。
- WhatsApp 自动导入。
- 多国家税务规则。
- AI 税务建议。

### 3.3 可选增强

- OCR 自动识别票据金额、日期、商家。
- 邮件转发收据到专属邮箱。
- Google Drive 导入。
- 自动生成租金收据 PDF。
- 月度提醒：未收租、缺票据、合同即将到期。

页面模块：

   - CTA：开始整理我的出租记录。
2. 痛点：
   - Excel 易坏。
   - 收据散落。
   - 税季混乱。
3. 产品截图或模拟 UI：
   - 月度租金。
   - 票据归档。
   - 年度税务包。
4. 适合谁：
   - LMNP。
   - 合租。
   - 1-5 套房。
5. 定价：
   - 简单买断或低价订阅。
6. FAQ：
   - 是否替代会计师：不替代。
   - 是否支持法国：第一版支持法国 LMNP 分类模板。

### 4.2 登录 `/login`

功能：

- 登录。
- 注册。
- 忘记密码。

注册时收集：

## 5. 数据模型

### 5.1 表结构总览

使用 Supabase Postgres。

核心表：

- `profiles`
- `workspaces`
- `workspace_members`
- `properties`
- `units`
- `tenants`
- `leases`
- `rent_charges`
- `rent_payments`
- `expenses`
- `documents`
- `tax_categories`
- `tax_exports`
- `subscriptions`

第一版虽然是单用户，也建议保留 `workspaces`，后续支持家庭成员或会计师查看时更容易。

### 5.2 profiles

```sql
profiles
- id uuid primary key references auth.users(id)
- email text
- full_name text
- default_workspace_id uuid
- locale text default 'fr'
- country_code text default 'FR'
- created_at timestamptz
- updated_at timestamptz
```

### 5.3 workspaces

```sql
workspaces
- id uuid primary key
- name text
- country_code text default 'FR'
- tax_regime text default 'LMNP'
- currency text default 'EUR'
- created_by uuid references profiles(id)
- created_at timestamptz
- updated_at timestamptz
```

### 6.4 权限模型

所有核心业务表都必须包含 `workspace_id`。

Row Level Security 策略：

- 用户只能访问自己所属 workspace 的数据。
- 第一版每个用户自动创建一个 workspace。
- 后续可以邀请会计师或家人加入 workspace。

### 6.5 文件存储

Supabase Storage bucket：

```text
documents
```

文件路径规则：

```text
workspace/{workspaceId}/documents/{year}/{documentId}-{safeFileName}
```

不要把用户原始文件名直接作为唯一文件路径，避免重名和特殊字符问题。

## 7. 税务与国家扩展设计

### 7.1 国家配置

用代码配置每个国家的基础规则：

```ts
type CountryConfig = {
  countryCode: 'FR' | 'US' | 'GB';
  defaultCurrency: 'EUR' | 'USD' | 'GBP';
  defaultLocale: string;
  supportedTaxRegimes: string[];
};
```

第一版：

```ts
FR: {
  countryCode: 'FR',
  defaultCurrency: 'EUR',
  defaultLocale: 'fr',
  supportedTaxRegimes: ['LMNP']
}
```

### 7.2 税务模板

税务模板按国家和制度区分：

```text
lib/tax/templates/fr/lmnp.ts
```

第一版模板只做分类、汇总和导出说明，不提供法律结论。

### 7.3 合规边界

产品内必须避免：

- “保证合规”。
- “自动完成报税”。
- “替代会计师”。

推荐表述：

- “Préparez vos justificatifs.”
- “Exportez un dossier clair pour votre comptable.”
- “Catégories indicatives pour LMNP.”

## 8. 开发阶段

### 阶段 0：项目初始化

目标：建立可部署的空壳。

任务：

1. 创建 Next.js + TypeScript 项目。
2. 配置 Tailwind。
3. 配置 next-intl。
4. 建立 `messages/fr.json`。
5. 创建基础路由。
6. 配置 ESLint/Prettier。
7. 部署到 Vercel。

验收：

- `/`、`/login`、`/dashboard`、`/settings` 可访问。
- 法语文案从 `messages/fr.json` 读取。

### 阶段 1：认证与账户

目标：用户可以注册、登录，并创建默认 workspace。

任务：

1. 创建 Supabase 项目。
2. 接入 Supabase Auth。
3. 创建数据库 migration。
4. 创建 `profiles`、`workspaces`、`workspace_members`。
5. 注册后自动创建 workspace。
6. 注册时保存国家和语言。
7. 添加 RLS。

验收：

- 新用户注册后进入 `/dashboard`。
- 数据库里有 profile 和 workspace。
- 用户无法访问其他 workspace 数据。

### 阶段 2：房源、房间、租客

目标：用户能建立出租结构。

任务：

1. 房源 CRUD。
2. 房间/单元 CRUD。
3. 租客 CRUD。
4. 租约 CRUD。
5. 房源详情页展示当前租约。

验收：

- 能创建一个合租房源和多个房间。
- 能把租客绑定到具体房间。
- 能记录月租、押金和租期。

### 阶段 3：租金记录

目标：按月份管理应收和实收。

任务：

1. 创建月度 rent charge 生成逻辑。
2. 手动添加付款。
3. 自动计算 unpaid/partial/paid 状态。
4. Dashboard 显示本月收租状态。
5. 房源详情显示月度表。

验收：

- 一份租约可以生成 12 个月应收。
- 部分付款状态正确。
- Dashboard 能看到本月未收租。

## 13. 风险与应对

### 13.1 税务规则复杂

应对：

- 第一版只支持法国 LMNP。
- 只做资料整理和分类，不做正式税务申报。
- 文案中建议用户交给会计师确认。

### 13.2 用户不愿持续订阅

应对：

- 支持一次性买断。
- 或年度税务模板包付费。
- 免费层让用户先录入数据，但导出收费。

### 13.3 上传文件后仍然混乱

应对：

- 文件必须有关联维度。
- 显示“未归档文件”队列。
- Dashboard 提醒缺少月份、房源或分类。

### 13.4 票据 OCR 成本

应对：

- 第一版 OCR 作为可选增强。
- 先让用户手动填金额和日期。
- 后续只对付费用户开放 OCR。
